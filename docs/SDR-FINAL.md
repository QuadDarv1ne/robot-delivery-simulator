# RTL-SDR Geoanalytics - Итоговая документация

## Что было реализовано

### 1. SDR Сервер (mini-services/sdr-server/index.ts)
- WebSocket сервер на порту 3004
- Эмуляция трёх протоколов: ADS-B, AIS, APRS
- Генерация спектральных данных (FFT 1024 точки)
- Трекинг истории перемещений объектов
- Реалистичная модель движения для Москвы

### 2. Интеграция с основным сервером (mini-services/robot-simulator-server/index.ts)
- Подключение к SDR серверу
- Ретрансляция SDR данных через порт 3003
- Новые команды управления SDR
- Автоматическое переподключение

### 3. SDR Панель (src/components/sdr/sdr-panel.tsx)
- Спектральный анализатор (1024 точки FFT)
- Waterfall-диаграмма (история 200 срезов)
- Статистика по типам сигналов
- Управление: режим, частота, усиление
- Список активных контактов с детализацией
- **Уведомления** о новых обнаружениях со звуком

### 4. Геоаналитика (src/components/sdr/geoanalytics.tsx)
- Карта Leaflet с маркерами объектов
- **Heatmap плотности** сигналов
- **Линии пути** перемещения объектов
- Фильтрация по **дальности от робота** (слайдер 1-100 км)
- Статистика обнаружений
- Режимы: карта/список

### 5. SDR Навигация (src/components/sdr/sdr-navigation.tsx)
- Панель в боковой панели симулятора
- **Автоматические предупреждения** о рисках
- Уровни риска: low/medium/high
- Ближайшие объекты по типам (ADSB, AIS, APRS)
- Проверка **безопасности пути**
- Статус общей безопасности

### 6. Хук для навигации (src/hooks/use-sdr-navigation.ts)
- `useSDRForNavigation()` - основной хук
- Расчет расстояний (формула гаверсинуса)
- Расчет пеленга
- Оценка уровня риска
- API для алгоритмов:
  - `getNearestADSB()` - ближайший самолёт
  - `getNearestAIS()` - ближайшее судно
  - `getNearestAPRS()` - ближайший маяк
  - `isPathClear()` - проверка пути
  - `getNearbyObjects()` - объекты в радиусе
  - `getAlerts()` - текущие предупреждения

### 7. UI Интеграция
- 2 новые вкладки: **SDR** и **Геоаналитика**
- Карточка SDR статуса в боковой панели
- **Панель SDR навигации**
- **Статус-бар** с информацией о сигналах
- **Звуковые уведомления** с разными тонами

## Запуск

```bash

npm run sdr           # SDR сервер (порт 3004)
```

## API для алгоритмов

```javascript
const { api } = useSDRForNavigation({ robotPosition })

// Получить ближайший ADS-B объект
const nearest = api.getNearestADSB()

// Проверить безопасность пути
const isClear = api.isPathClear(targetLat, targetLon, safetyRadius)

// Получить объекты в радиусе 5 км
const nearby = api.getNearbyObjects(5000)

// Получить все предупреждения
const alerts = api.getAlerts()
```

## Подключение реального RTL-SDR

Для подключения реального устройства:

1. Установите драйверы rtl-sdr
2. Запустите rtl_tcp
3. Замените эмуляцию в sdr-server/index.ts на вызовы к устройству
4. Используйте декодеры:
   - dump1090/readsb для ADS-B
   - gpsd/aisdecoder для AIS
   - direwolf/multimon-ng для APRS

## Расширение

Для добавления новых протоколов:

1. Добавьте тип в types.ts
2. Создайте генератор в sdr-server/index.ts
3. Добавьте режим в ModeSelector
4. Обновите статистику и UI
